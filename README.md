**University:** San Jose State University, Spring 2026
**Status:** Complete (Week 14/14)

## Overview

This project turns recent inference-efficiency research into a measured, reproducible
engineering case study on commodity student hardware (NVIDIA RTX 3050 Ti, 4 GB VRAM).
We evaluate attention-optimized kernels, KV-cache quantization, and their combination
to deliver practical guidance on what actually works under strict VRAM limits.

## Quick Start

### 1. Environment Setup

```bash
# Create conda environment
conda create -n llm_inference python=3.11 -y
conda activate llm_inference

# Install PyTorch with CUDA
pip install torch==2.6.0+cu124 --index-url https://download.pytorch.org/whl/cu124

# Install project dependencies
pip install -r requirements.txt

# Verify installation
python -c "import torch; print(f'PyTorch {torch.__version__}, CUDA: {torch.cuda.is_available()}')"
```

### 2. Running Experiments

```bash
# Run a single week
python scripts/run_all.py --week 1

# Run weeks 1-3
python scripts/run_all.py --from 1 --to 3

# Run all weeks sequentially
python scripts/run_all.py

# Run all weeks and generate plots
python scripts/run_all.py --plots

# Generate plots from existing results
python scripts/generate_plots.py
```

### 3. Running Individual Scripts

```bash
python scripts/week01_planning.py
python scripts/week02_setup_validation.py
python scripts/week03_baseline.py
# ... etc
```

## Results Summary

Key findings from 10 weeks of experiments on RTX 3050 Ti (4 GB VRAM):

- **FlashAttention-2** delivers the best OOM threshold improvement: **8064 tokens** vs 6784 baseline (+18.9%), enabling longer context on the same hardware
- **SDPA default** provides a modest OOM improvement to **7040 tokens** (+3.8%) with zero extra dependencies
- **KV-cache INT4 quantization** (via optimum-quanto) does not improve OOM threshold on this model/hardware combination, but reduces KV-cache memory per token
- **Combined configurations** (attention + KV-cache) are unstable on this hardware due to GQA head mismatch (16 query heads vs 2 KV heads)
- **Quality retention:** All stable configurations maintain accuracy within ≤2% of baseline on ARC-Easy, BoolQ, and HellaSwag
- **Practical recommendation:** Use FlashAttention-2 (via pre-built wheel) for maximum context length; use SDPA default as a zero-effort improvement

## Hardware Target

- **GPU:** NVIDIA RTX 3050 Ti Laptop (4 GB VRAM)
- **VRAM Budget:** ≤3.2 GB for model weights, ~0.8 GB headroom
- **Model:** Qwen/Qwen2.5-3B-Instruct (4-bit NF4 quantization)

## Benchmark Settings (Fixed)

| Parameter | Value |
|-----------|-------|
| Prompt lengths | 128, 512, 1024 tokens |
| Output limit | 64 tokens |
| Batch sizes | 1 |
| Decoding | Greedy (temperature=0) |
| Warmup runs | 2 |
| Benchmark runs | 5 |
| Seed | 42 |

## Metrics Collected

- **Latency:** p50, p95 (ms)
- **Throughput:** tokens/second
- **Memory:** Peak VRAM (MB), Peak CPU RAM (MB)
- **OOM Threshold:** Maximum context length before OOM
- **Correctness:** Top-1 token agreement rate, max logit difference
- **Quality:** 0-shot accuracy on ARC-Easy, BoolQ, HellaSwag (200 examples each)

## Success Criteria

At least one of:
- 20% reduction in p95 latency at a fixed context length
- 1.5x increase in maximum context length before OOM

While keeping quality retention within ≤2% accuracy drop.

## Troubleshooting

- **CUDA OOM:** Reduce batch size to 1, or reduce prompt length
- **bitsandbytes errors:** Ensure CUDA toolkit matches PyTorch CUDA version
- **FlashAttention unavailable:** Falls back to memory-efficient SDPA automatically
- **Thermal throttling:** Allow 1-2 min warmup; check GPU temp with `nvidia-smi`
- **Slow model download:** Set `HF_HOME` to a fast drive; models cache automatically
