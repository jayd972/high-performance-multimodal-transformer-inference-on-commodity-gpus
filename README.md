**Status:** Complete (Week 14/14)

## Overview

This project turns recent inference-efficiency research into a measured, reproducible
engineering case study on commodity student hardware (NVIDIA RTX 3050 Ti, 4 GB VRAM).
We evaluate attention-optimized kernels (including explicit FlashAttention-2 testing),
KV-cache quantization, and their combination across both text-only and multimodal
vision-language models to deliver practical guidance on what actually works under
strict VRAM limits.

## Quick Start

### 1. Environment Setup

```bash
# Create conda environment
conda create -n llm_inference python=3.11 -y
conda activate llm_inference

# Install PyTorch with CUDA
pip install torch==2.6.0+cu124 --index-url https://download.pytorch.org/whl/cu124

# Install project dependencies
pip install -r requirements.txt

# Verify installation
python -c "import torch; print(f'PyTorch {torch.__version__}, CUDA: {torch.cuda.is_available()}')"
```

### 2. FlashAttention-2 Installation (Optional but Recommended)

FlashAttention-2 is tested explicitly (not just as an SDPA fallback). Install it for
full benchmark coverage:

```bash
# Option A: Install from pre-built wheels (recommended for Windows)
# Visit https://github.com/mjun0812/flash-attention-prebuild-wheels
# Download the wheel matching your PyTorch + CUDA version, then:
pip install flash_attn-2.x.x-cp311-cp311-win_amd64.whl

# Option B: Build from source (Linux/WSL, requires CUDA toolkit)
pip install flash-attn --no-build-isolation

# Verify
python -c "import flash_attn; print(flash_attn.__version__)"
```

If FlashAttention-2 is not installed, all scripts gracefully fall back to SDPA.

### 3. Running Experiments

```bash
# Run all weeks for all models
python scripts/run_all.py

# Run a single week for a specific model
python scripts/run_all.py --week 3 --model qwen2.5-3b

# Run weeks 5-8 for multimodal models only
python scripts/run_all.py --from 5 --to 8 --model phi-3.5-vision llava-1.5-7b

# Run multimodal benchmarks only
python scripts/run_all.py --multimodal

# Run multimodal for a specific model
python scripts/run_all.py --multimodal --model phi-3.5-vision

# Run all weeks and generate plots
python scripts/run_all.py --plots
```

### 4. Running Individual Scripts

```bash
python scripts/week03_baseline.py --model qwen2.5-3b
python scripts/week05_attention_opt.py --model phi-3.5-vision
python scripts/week_multimodal.py --model llava-1.5-7b
```

## Results Summary

Key findings from experiments on RTX 3050 Ti (4 GB VRAM):

- **FlashAttention-2** delivers the best OOM threshold improvement: **8064 tokens** vs 6784 baseline (+18.9%), enabling longer context on the same hardware
- **FlashAttention-2 is tested explicitly** via the `flash-attn` package, not just as an SDPA sub-backend
- **SDPA default** provides a modest OOM improvement to **7040 tokens** (+3.8%) with zero extra dependencies
- **KV-cache INT4 quantization** (via optimum-quanto) reduces KV-cache memory per token
- **Combined FA2 + KV-cache** configurations are tested for MHA models (Phi-2, LLaVA, etc.)
- **Multimodal models** (Phi-3.5-vision, LLaVA-1.5-7B) are benchmarked with image+text inputs across all optimization axes
- **Quality retention:** All stable configurations maintain accuracy within ≤2% of baseline on ARC-Easy, BoolQ, and HellaSwag
- **Practical recommendation:** Use FlashAttention-2 for maximum context length; use SDPA default as a zero-effort improvement

## Metrics Collected

- **Latency:** p50, p95 (ms)
- **Throughput:** tokens/second
- **Memory:** Peak VRAM (MB), Peak CPU RAM (MB)
- **OOM Threshold:** Maximum context length before OOM
- **Correctness:** Top-1 token agreement rate, max logit difference
- **Quality:** 0-shot accuracy on ARC-Easy, BoolQ, HellaSwag (200 examples each)

## Attention Backends Tested

| Backend | Method | Notes |
|---------|--------|-------|
| `eager` | Standard PyTorch attention | Baseline, no optimization |
| `sdpa_default` | PyTorch SDPA (auto) | Zero-effort improvement |
| `sdpa_flash` | PyTorch SDPA flash backend | SDPA's flash sub-backend |
| `sdpa_mem_efficient` | PyTorch SDPA mem-efficient | SDPA's memory-efficient sub-backend |
| `sdpa_math` | PyTorch SDPA math fallback | Always available |
| `flash_attention_2` | flash-attn package (explicit) | Best OOM improvement |

## Success Criteria

At least one of:
- 20% reduction in p95 latency at a fixed context length
- 1.5x increase in maximum context length before OOM

While keeping quality retention within ≤2% accuracy drop.

## Troubleshooting

- **CUDA OOM:** Reduce batch size to 1, or reduce prompt length
- **bitsandbytes errors:** Ensure CUDA toolkit matches PyTorch CUDA version
- **FlashAttention-2 not installed:** Scripts fall back to SDPA automatically. Install from pre-built wheels for full coverage
- **Multimodal model OOM:** LLaVA-1.5-7B at 4-bit is ~3.5 GB — close to the 4 GB limit. Reduce prompt length if needed
- **Thermal throttling:** Allow 1-2 min warmup; check GPU temp with `nvidia-smi`
- **Slow model download:** Set `HF_HOME` to a fast drive; models cache automatically
